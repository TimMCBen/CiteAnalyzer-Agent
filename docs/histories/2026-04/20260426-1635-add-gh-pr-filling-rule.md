# 本次诉求

用户希望把创建远程 PR 时的执行要求明确写入仓库规则：优先使用本地 CLI；如果本地 CLI 不可用，要明确告知用户改为手动创建 PR；如果使用 `gh`，则需要按真实 diff 填写标题和正文。

## 主要改动

- 在 `CONTRIBUTING.md` 中新增 `使用本地 CLI 创建远程 PR 时的填写与兜底要求` 小节。
- 明确要求 Agent 在创建远程 PR 时先尝试本地 CLI；如果本地 CLI 不可用，要明确告知用户改为手动创建 PR。
- 明确要求 Agent 在使用 `gh` 创建 PR 时主动根据真实 diff 填写标题和正文，而不是只粘贴空模板。
- 明确 PR 标题应准确反映变更范围，不应直接使用分支名或模糊标题。
- 明确当分支范围超出本轮任务时，应先提示范围问题，再决定是否继续创建 PR。

## 设计动机

这条规则属于 PR 创建时的具体执行与兜底要求，放在 `CONTRIBUTING.md` 比放在仓库级协作原则文档中更合适，也更方便后续在实际创建 PR 时直接对照执行。

## 关键受影响文件

- `CONTRIBUTING.md`
