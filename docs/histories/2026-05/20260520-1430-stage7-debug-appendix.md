# Stage 7 报告主线与调试附录分离

## 用户诉求

用户希望报告页面更适合展示阅读，把多余的工程排查内容收进“调试附录”，不要让全量上下文、人工关注项和作者跳过明细干扰主报告。

## 主要改动

- 精简 Stage 7 HTML 导航，主报告只保留概览、核心指标、数据覆盖、分析摘要、可视化图表、重要学者和代表性引用。
- 新增默认折叠的“调试附录”，集中放置作者画像分类标准、作者画像跳过说明、机构分布、数据质量原始指标、系统关注项、全量引用上下文和补充发现。
- 将主区 `Data Quality` 调整为“数据覆盖情况”，用覆盖率卡片替代调试型原始质量指标。
- 将施引作者机构分布从主图表区移入调试附录，保留国家/地区地图作为主展示图。
- 将报告页面调整为更紧凑的两列布局：顶部左侧展示目标论文标题与链接，右侧展示导航和核心指标；主体左侧展示摘要、图表和代表性引用，右侧展示重要学者。
- 调试附录保持在页面最后一整行，避免打断主报告阅读路径。
- 将标题字号调小，避免长论文标题压迫首屏。
- 将作者相关图表拆成两张：`作者 h-index 分布` 按 `0-9 / 10-19 / 20-29 / 30-49 / 50-99 / 100+` 分桶展示影响力分布；`学者质量分布` 按重量级、高影响力、普通学者展示，并在柱子上标出数值。
- 微调主体两列宽度：右侧重要学者栏扩大到 `430px-500px`，左侧主列限制到 `920px`，并把中屏单列断点提高到 `1240px`，避免右侧表格显得拥挤。
- 重排首屏：左侧窄列集中展示论文标题、导航和核心指标，右侧宽列展示分析摘要；主体区不再重复展示摘要。
- 将数据覆盖情况并入分析摘要卡片下方，移除下方独立的数据覆盖区，让首屏摘要和覆盖率一起呈现。
- 统一首屏和主体区左列宽度：顶部标题/指标栏与下方图表左列都使用 `minmax(0, 920px)`，保证左侧信息栏与“引用年份趋势 / 引用情感分布”图表宽度对齐。
- 更新 Stage 7 合同测试，要求旧英文调试标题不再出现在页面中，并验证新调试附录结构。
