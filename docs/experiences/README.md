# 经验池索引

这里存放的是**协作经验池**，不是仓库长期规范。

经验池用于记录：

- 当轮任务中形成的可复用经验
- 还不适合直接升级为正式规则的过程知识
- 值得后续复盘和筛选的协作方法

正式机制见：

- `docs/EXPERIENCE_GOVERNANCE.md`

## 使用原则

1. **先记录，再决定是否升级**
2. **经验可以自动入池，但不自动立规矩**
3. **只有稳定复用后，才建议人工决定是否升级到正式规范**

## 推荐格式

每份经验池文档建议至少包含：

- 元信息
- 背景
- 一次结构化反思（可用 ralplan/ADR 形式）
- 经验条目
  - 标题 / 编号
  - 分类
  - 问 / 发生了什么
  - 答 / 经验是什么
  - 适用场景
  - 反例 / 失败模式
  - 是否建议升级
