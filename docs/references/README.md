# 外部参考资料

这个目录用于沉淀那些值得长期保留在仓库中的外部参考整理，方便后续开发时直接查阅。

这里放的不是执行计划、架构主文档，也不是大段复制的第三方原文，而是：

- 团队后续会反复参考的框架资料
- 当前项目会直接使用到的类、函数、能力清单
- 外部文档的筛选整理版
- 官方文档入口与最小可用说明

## 适合放这里的内容

- 框架参考
  - 例如 `LangChain`、`LangGraph`、Python 工程工具链
- API / 工具使用约定
  - 例如模型接入方式、工具封装入口、结构化输出能力
- 对外标准或合作方资料的简要整理

## 不适合放这里的内容

- 大段供应商文档原文
- 项目内部执行计划
- 仓库主架构设计
- 纯临时笔记
- 只在一次对话里用一次的草稿内容

## 使用方式

如果你在实现过程中遇到下面这些问题，优先先看这个目录：

- 这个框架里有哪些现成能力可以直接用？
- 当前项目更适合用哪些类、函数或能力？
- 官方文档入口在哪里？
- 哪些内容值得作为长期约定保留在仓库里？

如果内容已经超出“外部参考整理”的范围，比如开始变成：

- 项目自己的实现方案
- 当前阶段的执行计划
- 仓库真实目录结构

那就应该写到：

- `docs/product-specs/`
- `docs/ARCHITECTURE.md`
- `docs/exec-plans/`

而不是继续往 `references/` 里堆。
